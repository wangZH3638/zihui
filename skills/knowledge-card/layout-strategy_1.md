## 五、布局决策记录

### 为什么用三段式？
- 顶部：抓注意力，给主题定调
- 中部：承载核心内容，三栏形成节奏
- 底部：用金句收尾，给读者一个"带走"的 insight

### 为什么三栏？
- 三是信息的"黄金数"——足够信息量，不至于碎片化
- 三栏比两栏更有节奏感，比四栏更从容
- 数字锚点（01/02/03）强化阅读顺序

### 为什么 32/52/16 比例？
- 32% 顶部：标题有足够呼吸空间，不拥挤
- 52% 中部：内容是主体，需要最多的展示面积
- 16% 底部：引言条够醒目，但不至于抢戏

### 为什么金色作为强调色？
- 和深黑顶形成强对比
- 在白色背景上，金色数字锚点有"高光"感
- 暖色调给严肃内容增加温度

## 七、截图规范（已锁定）

### 截图流程
1. 设置 viewport = `820 × 1120`（比卡片大20px，避免边缘截到白边）
2. 截取 `.card` 元素
3. 自动裁剪：去除顶部和底部各 2px 浅色边框

### 最终尺寸
| 属性 | 值 |
|------|------|
| 画布 | 800 × 1067px |
| device_scale_factor | 2 |
| 输出分辨率 | 1600 × 2134px |
| 比例 | 3:4 |

### 关键实现
```python
# viewport 比卡片稍大
page = await browser.new_page(viewport={'width': 820, 'height': 1120}, device_scale_factor=2)
# 截取卡片
card = await page.query_selector('.card')
await card.screenshot(path=output_path)
# 后处理裁剪
img = Image.open(output_path)
img_cropped = img.crop((0, 2, w, h-2))  # 去除顶部底部各2px
img_cropped.save(output_path)
```
